# Відповіді на тест (backend теорія)

## 1

``` php
$a = new ClassA();
$b = new ClassA();
var_dump($a === $b);
```

**Відповідь:** `bool(false)`\
(різні об'єкти → різні посилання)

## 2

``` php
[...] === [...]
```

**Відповідь:** `bool(true)`\
(масиви ідентичні за структурою та значеннями)

## 3

``` php
$b = $a;
$b->var1 = 10;
echo $a->var1;
```

**Відповідь:** `10`\
(об'єкти передаються за посиланням)

## 4

``` php
$b = $a;
$a->var1 = 10;
$a = null;
echo $b->var1;
```

**Відповідь:** `10`\
(об'єкт існує, поки є посилання `$b`)

## 5

``` php
$b = &$a;
$b->var1 = 10;
$a = null;
echo $b->var1;
```

**Відповідь:** помилка\
(змінна стала `null`, об'єкт втрачено)

## 6

``` php
foreach ($numbers as $number) {
    $number **= 2;
}
echo $numbers[4];
```

**Відповідь:** `5`\
(ітерація без посилання --- масив не змінюється)

## 7

``` php
foreach ($numbers as &$number) {
    $number **= 2;
}
$number = 10;
echo $numbers[4];
```

**Відповідь:** `10`\
(останній елемент лишається прив'язаним посиланням)

# SQL частина

## 16

``` sql
SELECT
    a.id AS article_id,
    a.title AS article_title,
    a.body AS article_body,
    c.id AS comment_id,
    c.title AS comment_title,
    c.body AS comment_body,
    c.parent_id AS comment_parent_id
FROM articles a
JOIN comments c ON c.article_id = a.id
WHERE a.id = 101;
```

## 17

``` sql
SELECT
    u.id AS user_id,
    u.email,
    p.first_name,
    p.last_name,
    p.photo_link,
    r.title AS role_title
FROM users u
LEFT JOIN profiles p ON p.user_id = u.id
LEFT JOIN users_roles ur ON ur.user_id = u.id
LEFT JOIN roles r ON r.id = ur.role_id
WHERE u.id = 256;
```
